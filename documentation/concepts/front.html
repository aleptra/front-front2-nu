Front is the low-code frontend framework for building interactive, data-driven, and modular applications using a markup-first approach. Unlike traditional frontend frameworks or libraries that rely heavily on complex logic, Front keeps development simple, clean, and maintainable. Everything can be expressed through markup, enabling developers and designers to work seamlessly together while producing readable and scalable code.

Core Ideas

  • Markup-first — express behavior directly in markup.
  • Attribute DSL — FTML maps attributes to actions, modules, and plugins.
  • Deterministic runtime - explicit ordering via runorder, predictable execution.
  • No build step — works from a CDN or any static host.
  • Extensible — load modules/plugins at runtime.

Runtime Pipeline

  • Load runtime → discover module/plugin/var from the script tag.
  • Fetch & initialize modules/plugins; read configuration from attributes (e.g., *-conf).
  • Scan DOM → map FTML attributes to actions; respect runorder and stop/start.
  • Execute actions directly on the DOM; preserve original state for reversibility.
  • Wire events (click, input, etc.); re-run scopes when needed (rerun).

Document Object Model (DOM)

Front operates directly on the real DOM rather than using a virtual DOM abstraction. This approach provides several advantages:

  • Performance: Updates are applied immediately without diffing overhead.
  • Memory Efficiency: No large virtual DOM copy is maintained in memory.
  • Reversibility: Original DOM state is preserved, allowing deterministic updates and easy rollbacks.
  • Predictability: Combined with the deterministic runtime, changes happen exactly as specified in markup attributes.

Partial Page Application (PPA)

Module Required: Navigate

Partial Page Application (PPA) behavior intercepts link clicks and dynamically loads content into a target container without a full page reload. Navigation is controlled via attributes on links or elements, enabling seamless updates using AJAX.

Note: There is no route table - just link to separate .html files.

State Binding

Front keeps application state in sync across the page. Variables, user inputs, query parameters, global values, and API responses are automatically connected to page elements so that any changes are reflected wherever they are used.

Persistent State Binding

Requires: Storage module

Front also supports persistent state binding via the storage module, which keeps bound values saved across page reloads. Changes in storage automatically update all connected elements, ensuring data remains consistent between sessions

Dynamic Data Rendering

Requires: Data module

Front provides a way for fetching, binding, and rendering data directly in your page. It allows content to be updated dynamically based on external data sources, supporting lists, filtering, sorting, and merging of data.

Event Action Handling

Front lets you attach events directly to elements in markup. Built-in and custom events trigger actions automatically, enabling interactive behavior and dynamic updates across the page.

Globalization

Requires: Internationalize module, Chronitize module and Monetize module

With the globalize module, mark strings using globalize-get. Language can be switched via querystring (?locale=sv).
