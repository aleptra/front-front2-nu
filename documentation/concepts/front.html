Front is the low-code frontend framework for building interactive, data-driven, and modular applications using a markup-first approach. Unlike traditional frontend frameworks or libraries that rely heavily on complex logic, Front keeps development simple, clean, and maintainable. Everything can be expressed through markup, enabling developers and designers to work seamlessly together while producing readable and scalable code.

Essentials:

  • Markup-First Development: Define all UI elements and behavior directly in markup.
  • Interactive and Dynamic: Built-in attributes enable events, data-binding, and reactive behavior.
  • Modular Architecture: Extend markup functionality with modules or plugins.
  • Template-Driven Layouts: Reusable templates ensure consistency across application.
  • Instant Prototyping: Runs natively in the browser-no compilers or build tools required.
  • Minimal Learning Curve: HTML-like syntax makes it approachable for both designers and developers.
  • AI-Friendly: Structured declarative markup makes AI code generation easier.
  • Deterministic Runtime: Actions occur in a predictable order using runorder.
  • No Build Step: Works immediately from a CDN or any static host.
  • Attribute DSL: FTML maps attributes to actions, modules, and plugins.

Runtime Pipeline

  • Load runtime → discover module/plugin/var from the script tag.
  • Fetch & initialize modules/plugins; read configuration from attributes (e.g., *-conf).
  • Scan DOM → map FTML attributes to actions; respect runorder and stop/start.
  • Execute actions directly on the DOM; preserve original state for reversibility.
  • Wire events (click, input, etc.); re-run scopes when needed (rerun).

Document Object Model (DOM)

Front operates directly on the real DOM rather than using a virtual DOM abstraction. This approach provides several advantages:

  • Performance: Updates are applied immediately without diffing overhead.
  • Memory Efficiency: No large virtual DOM copy is maintained in memory.
  • Reversibility: Original DOM state is preserved, allowing deterministic updates and easy rollbacks.
  • Predictability: Combined with the deterministic runtime, changes happen exactly as specified in markup attributes.

Core features

- The fundamental behavior built into the engine.

Runtime Hybrid

Actions occur in a determined order when using the runorder attribute, ensuring consistent updates and reducing side effects or race conditions, while the natural element order is respected when no explicit runorder is set.

Action Chaining

Multiple actions can be chained on a single element through attribute values. Each attribute triggers an action, and together they form a declarative execution chain processed by the runtime. This allows complex behavior to be built by stacking small, reusable actions.

Event Action Handling

Events are triggered through markup attributes. Attributes that support events invoke corresponding actions, allowing elements to respond immediately and update content, control modules, or modify other elements declaratively.

Dynamic Content

Update element content in real time as data or user actions change. Declarative attributes automatically sync visibility, formatting, and lists, ensuring the interface stays current without the need for manual structural changes.

State Binding

Page elements stay in sync with application state. Variables, user inputs, query parameters, global values, and API responses update automatically wherever they appear.

Conditional Execution

Allows actions or attributes to run only when specified conditions are met. Elements can evaluate values, bindings, or expressions to control whether updates occur, enabling declarative logic within the page.

Execution Control

Manage the lifecycle of your interface with total flexibility. Keep elements dormant to prevent initialization and logic execution until they are required. Activating these sections dynamically on demand ensures resources are utilized only when relevant, maximizing efficiency.

Scope Management

Perform surgical UI updates by re-evaluating specific components or containers. This localized approach allows you to refresh individual parts of your interface instantly without a full page reload, keeping your application fast and memory-efficient.

Recursive Component Composition

Build modular interfaces by stacking external files and internal fragments. Using a layered injection system, you can override templates indefinitely-later fragments extend or replace earlier ones-allowing for deeply nested, reusable UI components that remain fully scoped and reactive.

Modular Includes

Seamlessly inject external HTML templates into your page using a single attribute. This allows you to manage reusable components like headers, footers, or UI fragments in separate files, keeping your main markup clean and easy to maintain.

Module features

- The extended behavior loaded into the engine.

Single Page Application (SPA)

Module: Navigate

Intercept link clicks and dynamically loads content into a target container without a full page reload. Navigation is controlled via attributes on links or elements, enabling seamlessupdates using AJAX, with links pointing directly to separate .html files instead of a route table.

Dynamic Data Rendering

Module: Data

Fetch, bind, and render dynamic data directly into pages. Content can be updated automatically from external sources, with support for lists, filtering, sorting, merging, transforming, and caching. Built-in lifecycle hooks and other features make it flexible for interactive, data-driven interfaces.

Persistent State Binding

Module: Storage

Persistent state binding keeps values saved across page reloads and automatically updates all connected elements. It supports structured data, enabling adding, updating, setting, or removing values, and allows stored data to be injected directly into element content or attributes, keeping the interface synchronized and reactive.

Globalization

Module: Internationalize, Chronotize and Localize

Ensure that interfaces adapt seamlessly to users worldwide. Support multilingual, regional formatting of dates, times, numbers, and other content, as well as proper handling of locale-specific conventions. Combining translation, localization, and temporal adaptation allows the interface to respond accurately and consistently across different languages and regions.

Responsive Breakpoint Actions

Module: Screen

Track window size and automatically apply element-specific logic based on breakpoints. Detect viewport changes and ensures that elements respond dynamically to different screen widths, keeping the interface adaptable and consistent across devices.

Keyboard Action Handling

Module: Keyboard

Listens for key presses and sequences, allowing elements to trigger actions when specific keys or words are typed. Supports single-key and word sequences, scoped behavior, and custom finalizers, enabling fully declarative, keyboard-driven interactions within the interface.

Animate

Module: Animate

Enables declarative animations for elements with configurable keyframes, transforms, duration, easing, delay, direction, and iteration. Animations can be controlled dynamically.
