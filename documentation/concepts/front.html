Front is the low-code frontend framework for building interactive, data-driven, and modular applications using a markup-first approach. Unlike traditional frontend frameworks or libraries that rely heavily on complex logic, Front keeps development simple, clean, and maintainable. Everything can be expressed through markup, enabling developers and designers to work seamlessly together while producing readable and scalable code.

Core Ideas

  • HTML-first — express behavior directly in markup.
  • Attribute DSL — FTML maps attributes to actions, modules, and plugins.
  • Deterministic runtime - explicit ordering via runorder, predictable execution.
  • No build step — works from a CDN or any static host.
  • Extensible — load modules/plugins at runtime.

Runtime Pipeline

  1. Load runtime → discover module/plugin/var from the script tag.
  2. Fetch & initialize modules/plugins; read configuration from attributes (e.g., *-conf).
  3. Scan DOM → map FTML attributes to actions; respect runorder and stop/start.
  4. Execute actions directly on the DOM; preserve original state for reversibility.
  5. Wire events (click, input, etc.); re-run scopes when needed (rerun).

Extensions

Front supports a modular architecture: load functionality as modules or plugins at runtime. This page uses globalize and data via the script tag’s module attribute. 

<script src="https://cdn.front.nu/nightly/front.js" module="globalize;data"></script>

State Management

Front favors simple, explicit state: bind variables into attributes/content and update them via inputs or actions. The runtime updates affected nodes deterministically.

Event Handling

Bind events declaratively with the click attribute (and others). Actions can target the current element (#) or any selector.

Globalization

With the globalize module, mark strings using globalize-get. Language can be switched via querystring (?locale=sv, for example) depending on your locale setup.

Framework

Markup-first and modular

Developer Experience

Front emphasizes simplicity: drop a script, author declaratively in HTML, and iterate quickly. Its clear attribute semantics minimize boilerplate and keep mental overhead low.
