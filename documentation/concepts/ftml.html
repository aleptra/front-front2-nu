FTML (Front Text Markup Language) is the primary language of Front. It extends standard HTML with a declarative layer that makes it possible to define structure, style, layout, interactivity, and behavior entirely through attributes, eliminating the need for any language other than FTML.

Fundamentals

FTML is structured around a small set of fundamentals that define how interfaces are described and composed: Attributes, Variables, Pages, Templates, Modules, and Plugins. These elements provide structure, state, reuse, and extensibility directly in markup, forming a cohesive model for building declarative and reactive interfaces.

Low-Code

FTML embodies a Low-Code approach, allowing developers and designers to define structure, styling, behavior, and interactivity directly through markup attributes. Loops, conditionals, computed values, event handling, and reusable templates are all expressed declaratively, reducing boilerplate, accelerating prototyping, and promoting rapid, maintainable, and consistent application development within a single markup language.

The Mental Model

Each attribute expresses a small unit of behavior. Together, they form a graph describing the interface. Developers focus on intent rather than components or lifecycle, thinking in terms like:

  • "This should load data."
  • "This should react when a value changes."
  • "This should appear conditionally."
  • "This should map to a global value."
  • "This should repeat for each item."

This mental model simplifies reasoning and makes interfaces self-describing, enabling a direct path from intent to implementation.

FTML vs Traditional Markup

FTML simplifies interface development by overcoming limitations of traditional markup languages like HTML, XHTML, and XML, which often lack dynamic capabilities and require external scripts.

Comparison of FTML with Traditional Markup:
Feature Traditional Markup FTML
Structure ❌ Static ✅ Static + dynamic
Interpretation ❌ Browser renders static markup ✅ Browser renders compiled output
Logic ❌ Requires external scripts ✅ Declarative logic in markup
Styling ❌ Separate CSS stylesheets ✅ Declarative styling in markup
Dynamic content ❌ Requires scripts and templating ✅ Bindings & loops in markup
Interactivity ❌ JavaScript event handlers ✅ Declarative event handling
Components ⚠️ Manual reuse via includes or scripts ✅ Reusable components
Reactive updates ❌ Manual DOM manipulation ✅ Auto DOM sync
Templates ⚠️ Separate template engines ✅ Reusable templates in markup
Computed values ❌ Computed in scripts ✅ Computed in markup
Data-driven rendering ❌ Requires scripting ✅ Reactive rendering
Conditionals ❌ Scripted conditionals ✅ Conditional rendering in markup
Event handling ⚠️ Inline or external JavaScript ✅ Inline declarative events
