Introduction

FTML (Front Text Markup Language) is a declarative, Low-Code markup language that integrates structure, styling, and behavior directly within markup. It enables efficient creation of dynamic, reactive, and reusable web interfaces with minimal boilerplate, streamlining development and collaboration between developers and designers.

Low-Code

FTML embodies a Low-Code approach by allowing developers and designers to define application structure, styling, behavior, and interactivity directly through markup attributes. Common patterns like loops, conditionals, computed values, event handling, and reusable templates are all expressed declaratively without writing JavaScript or CSS, reducing boilerplate and accelerating prototyping.

This attribute-driven, declarative syntax enables rapid development, consistent application behavior, and maintainable code while allowing teams to collaborate seamlessly within a single markup language.

FTML vs Traditional Markup

FTML extends traditional markup languages like HTML, XHTML, and XML by embedding declarative logic, styling, and interactivity directly within markup. Unlike conventional markup, which requires separate scripts and stylesheets for dynamic behavior, FTML allows developers to express complex, data-driven interfaces directly in markup.

This declarative, Low-Code approach reduces boilerplate, streamlines collaboration between developers and designers, and ensures maintainable and scalable code. Features like reactive bindings, templates, computed values, and inline event handling differentiate FTML from standard markup languages.
