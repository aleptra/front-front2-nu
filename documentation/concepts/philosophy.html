Philosophy

The project began in 2019 by Josef Gabrielsson to address the growing complexity of interface development across platforms. From the beginning, the idea of expressing application behavior directly in markup was met with skepticism. Many questioned whether markup could responsibly handle concerns traditionally reserved for imperative code. To ensure the paradigm was sound, the project remained closed for several years while the declarative model was explored, stress-tested, and refined before its public release.

The Challenge

Traditional approaches introduced mental overhead through layered abstractions and rigid structures. Logic, styling, and data were separated across contexts, forcing developers to shift syntaxes to express a single idea. A single interaction often had to be translated between multiple conceptual layers. This fragmentation obscured intent and made interfaces harder to reason about.

Core Principles

  • Declarative: Describe what happens, not how; the runtime handles it.
  • Markup as Runtime: Markup expresses structure, behavior, and styling directly.
  • Predictable: Actions occur in order, making interfaces easy to reason about.
  • Minimal Ceremony: Works immediately without build steps or complex toolchains.
  • Clarity: Logic and structure prioritize comprehension over patterns.
  • Extensible: Modules and plugins add behavior without complexity.
  • Minimal Logic: Logic stays concise and close to markup.
  • Universal: Compatible across platforms while enhancing markup.

A Different Mental Model

In Front, each attribute expresses a small unit of behavior. Together, they form a graph describing the interface. Developers focus on intent rather than components or lifecycle, thinking in terms like:

  • "This should load data."
  • "This should react when a value changes."
  • "This should appear conditionally."
  • "This should map to a global value."
  • "This should repeat for each item."

This mental model simplifies reasoning and makes interfaces self-describing, enabling a direct path from intent to implementation.
