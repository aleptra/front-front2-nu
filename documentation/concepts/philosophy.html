Philosophy

Front, powered by FTML, is built on a declarative, low-code, markup-first philosophy that unifies structure, behavior, styling, and data. Developers focus on what the interface should express, while the runtime handles details seamlessly.

Origin Story

Front and FTML began in 2019 by Josef Gabrielsson to address growing complexity in interface development across platforms. Traditional frameworks scattered logic and styling across multiple layers, obscuring the core goal: to build interfaces. The project was intentionally kept closed for several years while the declarative paradigm was explored and validated, allowing the approach to mature before being released publicly.

The Challenge

Traditional approaches imposed mental overhead through scattered abstractions, rigid structures, and unnecessary steps. Logic, styling, and data handling were often separated across multiple layers, requiring developers to move between different syntaxes and contexts to express a single idea. Developers were forced to translate intent into framework-specific patterns and boilerplate. This fragmentation reduced clarity, increased context-switching, and made even simple interfaces harder to reason about and maintain.

The Solution

Front introduces a new paradigm in interface development: a declarative, low-code approach that unifies structure, behavior, styling, and data. It minimizes friction, making interfaces intuitive and approachable while remaining powerful.

Front provides a declarative framework where markup is the authoritative source of truth. Every element expresses intent, forming a cohesive, self-describing runtime.

Guided by a modular, declarative philosophy, components and plugins manage responsibilities, letting developers focus on high-level design and creating clear, coherent experiences.

Foundational Pillars

1. Declarative Expression

Developers describe what should happen, not how. The runtime interprets intent automatically.

2. Markup as the Runtime

Markup serves as the expressive surface for declarative structure, behavior, and styling across platforms, letting developers build interfaces directly in markup without writing JavaScript or CSS.

3. Predictable and Reversible

Actions occur in a clear, deterministic order, preserving clarity and making the interface easy to reason about.

4. Minimal Ceremony

No toolchains or compilation are required. Interfaces work immediately in any environment.

5. Pragmatic Clarity

Understanding is prioritized over rigid patterns. Logic is placed where it enhances comprehension.

6. Extensible Simplicity

Modules and plugins extend behavior without adding complexity, keeping the runtime transparent and approachable.

7. Minimalistic Logic

Logic is kept concise and close to the markup it affects, avoiding unnecessary abstraction.

8. Universal Compatibility

Works reliably across platforms while enhancing markup rather than replacing it.

A Different Mental Model

In Front, each attribute expresses a small unit of behavior. Together, they form a declarative graph describing the interface. Developers think in terms of intent rather than components or lifecycle.

  • "This should load data."
  • "This should react when a value changes."
  • "This should appear conditionally."
  • "This should map to a global value."
  • "This should repeat for each item."

This mental model simplifies reasoning and makes interfaces self-describing.

The Guiding Idea

Building interfaces should feel like crafting a document, not assembling a machine.

By collapsing unnecessary layers and embracing declarative expression, Front creates a direct path from intent to implementation. Developers write markup that communicates meaning, which the runtime interprets seamlessly.

The result is a framework that is powerful yet approachable, clear, and aligned with the principles of intuitive, declarative interface design.
