Attributes are the core of FTML: they declaratively define behavior, style, content, interactivity, and logic directly in markup. Everything in FTML is expressed through attributes, making your code readable, maintainable, and fully declarative.

Attribute Layers

  • Core Attributes: The essential FTML attributes that define element behavior, appearance, and logic.
  • Module Attributes: Optional attributes that extend core capabilities with modular features.
  • Plugin Attributes: Optional attributes for custom behaviors or third-party integrations.

Execution order

Natural Order

By default, attributes run sequentially in the order they appear. When multiple attributes affect the same property, the last one takes precedence.

Example <!-- Element displays "Second text" because the last settext wins --> <div settext="First text" settext="Second text"></div>

Selective Order

The default sequence can be overriden with the runorder attribute to define the exact execution order for attributes.

Example <!-- Attribute execution order: settext → uppercase → lowercase --> <div runorder="uppercase;lowercase" settext="hello world" lowercase uppercase></div>

Best Practices

  • Use attributes declaratively, not imperatively.
  • Group related attributes for clarity.
  • Don't use runorder unless necessary.
  • Keep logic simple; delegate complex behavior to modules or plugins.
