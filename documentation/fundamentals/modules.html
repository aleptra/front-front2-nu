Modules are self-contained packages that extend the core functionality of your application. They provide additional features, integrations, or behavioral enhancements without modifying the base system. They are included in the Front library and promote modularity, scalability, and a clean separation of concerns by allowing developers to enable or disable them as needed.

Activating Modules

Modules are loaded via the module attribute of the <script> element. You can activate one or multiple modules in your application.

FTML (index.html) <!doctype html> <html>  <head>   <!-- Load Front with multiple modules (navigate and animate) -->   <script     src="https://cdn.front.nu/nightly/front.js"     module="navigate;animate">   </script>  </head>  <body>   <main>    <h1>Modules Example</h1>   </main>  </body> </html>
Note: Separate multiple modules using semicolons (;). Avoid using commas or spaces.

Configuring Modules

Each module can define its own configuration parameters using the {module}-conf attribute of the <script> element. You can pass initialization settings such as enabling debug mode, defining routes, or adjusting animation behavior.

FTML (index.html) <script  src="https://cdn.front.nu/nightly/front.js"  module="navigate;animate"  navigate-conf="transition:'fade',speed:300"  animate-conf="duration:500,easing:'ease-in-out'"> </script>
Note: Configuration attributes follow the pattern {module}-conf and must be placed on the same <script> element that loads Front.
