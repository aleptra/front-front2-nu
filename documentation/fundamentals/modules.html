Modules are self-contained packages that extend the core functionality of your application. They provide additional features, integrations, or behavioral enhancements without modifying the base system. Modules promote modularization, scalability, and clean separation of concerns by allowing developers to enable or disable features as needed.

Each module can define its own attributes, configuration settings, and events, seamlessly integrating with the Front core or other modules. Modules can be included dynamically in your application and are initialized automatically when Front loads.

Including Modules

Modules are loaded via the module attribute of the <script> element. You can include one or multiple modules in your application. When specifying multiple modules, separate each name with a semicolon (;).

Example:

index.html <!doctype html> <html>  <head>   <!-- Load Front with multiple modules (navigate and animate) -->   <script     src="https://cdn.front.nu/nightly/front.js"     module="navigate;animate"     conf="debug:true;">   </script>  </head>  <body>   <main>    <h1>Modules Example</h1>   </main>  </body> </html>
Note: Separate multiple modules using semicolons (;). Avoid using commas or spaces.

Configuring Modules

Each module can define its own configuration parameters using the ...-conf attribute of the <script> element. You can pass initialization settings such as enabling debug mode, defining routes, or adjusting animation behavior.

Example:

index.html <script  src="https://cdn.front.nu/nightly/front.js"  module="navigate;animate"  navigate-conf="transition:'fade',speed:300"  animate-conf="duration:500,easing:'ease-in-out'"> </script>
Note: Configuration attributes follow the pattern {module}-conf and must be placed on the same <script> element that loads Front.
