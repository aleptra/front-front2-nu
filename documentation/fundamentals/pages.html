Pages represent structured, self-contained views of your application. They act as containers for elements, layouts, and FTML attributes, defining how content is presented, styled, and behaves. As the building blocks of your application, pages help organize your app into logical sections or screens and can be navigated to directly. Different types of pages exist, each tailored to specific purposes, providing modular and reusable views for sections, features, or workflows.

Start Page

A Start Page is the entry point of your application. It is loaded first, sets up the environment, and serves as a primary template, providing configurations, scripts, styles, and layout structures that other pages, templates, modules, and plugins can rely on.

Responsibilities:

  • Load core attributes, modules, plugins and assets.
  • Set up configurations.
  • Define the default page layout and structure.

Requirements:

  • Page must include the <!doctype html> declaration at the top to define the page as a Start Page.
  • Front library must be included inside the <head> tag using src attribute of the <script> tag, either by loading the CDN or by hosting front.js yourself.

Optionals:

The following elements can optionally be defined inside the <head> tag:

  • Configuration can be set using conf (core), ...-conf (modules), and ...--conf (plugins) attributes of the <script> tag.
  • Variables can be set using var attribute of the <script> tag.
  • Base href can be set using hrefhost attribute of the <base> tag.
  • Page title can be set using the <title> tag.
  • Metadata can be set using the <meta> tag.
  • Favicons, icons, and resource hints can be set using <link> tags.

Example:

index.html <!doctype html> <html>  <head>   <base hrefhost="front.nu:/">   <script     src="https://cdn.front.nu/nightly/front.js"     var="api"     module="keyboard"     conf="debug:true;">   </script>   <title>My App Start Page</title>  </head>  <body>   <main height="100%" include="_includes/home.html"></main>  </body> </html>

Standard Page

A Standard Page is a flexible page that builds upon the base structure of your application. It can override elements, layouts, or attributes defined in a StartPage, allowing you to customize the content, style, and behavior for specific sections or workflows. Standard Pages act as containers for elements, layouts, and FTML attributes, providing modular and reusable views that help organize your application into logical sections or screens.

Responsibilities:

  • Inherit and optionally override elements, layouts, and configurations from the Start Page.
  • Organize and present content specific to a section, feature, or workflow.
  • Ensure consistency with the application's overall structure and style.
