Pages represent structured, self-contained views of your application. They act as containers for elements, layouts, and FTML attributes, defining how content is presented, styled, and behaves. As the building blocks of your application, pages help organize your app into logical sections or screens and can be navigated to directly. Different types of pages exist, each tailored to specific purposes, providing modular and reusable views for sections, features, or workflows.

Start Page

A Start Page is the entry point of your application. It sets up the environment by loading core attributes, extended attributes via modules and plugins, and allows defining variable assets that can be used throughout the application. It provides the primary page and layout base that Standard Pages can rely on, and supports global configuration to control the functionality and behavior of the application.

Requirements:

  • Page must include the <!doctype html> declaration at the top to define the page as a Start Page.
  • Front library must be included inside the <head> element using src attribute of the <script> element, either by loading the CDN or by hosting front.js yourself.

Optionals:

The following elements can optionally be defined inside the <head> element:

  • Configuration can be set using conf (core), ...-conf (modules), and ...--conf (plugins) attributes of the <script> element.
  • Variables can be set using var attribute of the <script> element.
  • Base URL can be set using hrefhost attribute of the <base> element.
  • Page title can be set using the <title> element.
  • Metadata can be set using the <meta> element.
  • Favicons, icons, and resource hints can be set using <link> elements.

Example:

index.html <!doctype html> <html>  <head>   <!-- Set the base URL according to the host for relative links -->   <base hrefhost="front.nu:/">   <!-- Load Front with the navigate module and debug enabled -->   <script     src="https://cdn.front.nu/nightly/front.js"     module="navigate"     conf="debug:true;">   </script>   <!-- Set the page title -->   <title>Start Page</title>  </head>  <body>   <header></header>   <main>    <!-- Link to standard page -->    <a href="page1.html">Go to Standard Page</a>   </main>   <footer></footer>  </body> </html>
Note: Set the base URL correctly to match your host.

Standard Page

A Standard Page is a regular page that can work independently or inherit settings, configurations, modules, and plugins from a Start Page. It follows the same structural conventions, ensuring consistency across the application. When linked to a Start Page, it extends its setup while allowing its own content and layout to override or supplement what is inherited.

Requirements:

  • Page must include the <script> element with the Front library to ensure initialization and compatibility.

Optionals:

  • Templates can be loaded by including a <template> element inside the <head> element, using srcdoc or src attributes to link the source.
  • Page title can be set using the <title> element.
  • Content inside the <body> element can replace what is defined in the Start Page.

Example:

page1.html <html>  <head>   <script src="https://cdn.front.nu/nightly/front.js"></script>   <!-- Loads two templates:    1. Start Page (srcdoc="./index") — base layout    2. Layout template (src="./_templates/template") — overrides base layout -->   <template srcdoc="./index" src="./_templates/template1"></template>   <!-- Set the page title -->   <title>Standard Page</title>  </head>  <body>   <h1>Hello world!</h1>  </body> </html>
Note: Ensure template paths are correct and do not include file extensions.
