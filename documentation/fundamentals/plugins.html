Plugins are third-party components that extend the capabilities of your application by adding new features, integrations, or custom behavior. They can modify or enhance modules without changing the core system, allowing developers to easily add functionality and maintain a modular architecture.

Loading Plugins

Plugins are loaded via the plugins attribute of the <script> element. You can activate one or multiple modules in your application.

Example:
FTML (index.html) <!doctype html> <html> <head> <!-- Load Front with multiple plugins (chat and analytics) --> <script src="https://cdn.front.nu/nightly/front.js" plugin="chat;analytics" conf="debug:true;"> </script> </head> <body> <main> <h1>Plugins Example</h1> </main> </body> </html>
Note: Separate multiple plugins using semicolons (;). Avoid using commas or spaces.

Configuring Plugins

Each plugin can define its own configuration parameters using the {plugin}-conf attribute of the <script> element. You can pass settings such as enabling debug mode, API keys, theme options, or callback functions.

Example:
FTML (index.html) <script src="https://cdn.front.nu/nightly/front.js" plugin="chat;analytics" chat-conf="theme:'dark',position:'bottom-right'" analytics-conf="trackingId:'UA-123456',debug:true"> </script>
Note: Configuration attributes follow the pattern {plugin}-conf and must be placed on the same <script> element that loads Front.
