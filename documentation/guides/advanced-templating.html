This guide explains how to use advanced templating features to create flexible and reusable templates.

Define Template File

Best practice:

  • Template should be wrapped inside a <template> element.
  • No other tags should be outside the <template> element.
  • Store template files inside a _templates folder at the root of your project.

Example:

FTML (_templates/template1.html) <!-- Defines a reusable layout --> <template> <header></header> <aside></aside> <main include></main> <aside></aside> <footer></footer> </template>
Note: If an FTML attribute is without a value, it will be removed from the final output.

Load Template in a Standard Page

Once you have defined a template file (e.g., template1.html), you can load it into a Standard Page using the <template> element and src attribute.

Important:

  • src attribute, which points to a layout template, requires a srcdoc attribute.
  • srcdoc attribute defines the base Start Page layout, and src will override or inject content into it.

Example:

FTML Usage <!-- Loads two templates: 1. Start Page (srcdoc="./index") - base layout 2. Layout template (src="./_templates/template1") - overrides base layout --> <template srcdoc="./index" src="./_templates/template1"></template>
Note: Ensure template paths are correct and do not include file extensions.
