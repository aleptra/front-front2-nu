This guide explains how to build a plugin for Front using ECMA5 JavaScript and load it with FTML.

To build a plugin, you need:

  • Choose a unique plugin name that is not already in use.
  • Load Front with your plugin name(s) via the plugin attribute on the script tag.
  • Create a plugin file in the plugins/ directory.
  • Optionally configure the plugin using {plugin}-conf on the script tag to pass settings like themes, API keys, or debug options.

Plugin Structure

Each plugin is defined as a property on app.plugin with your chosen plugin name. A plugin can export any number of methods. There are two special conventions:

  • __autoload: Called automatically when the plugin is loaded; receives an options object.
  • pluginName--methodName: Attributes matching this pattern invoke the corresponding method with an options object (including element).

Plugin methods can manipulate the DOM, handle events, or interact with the API.

Example Plugin:

Below is a minimal example plugin that logs when loaded and provides a function to update an element.

JS (example.js) 'use strict'; app.plugin.example = { __autoload: function(options) { app.log.info()('Example plugin loaded!', options && options.name) }, highlight: function(options) { var el = options.element if (el) { el.style.background = '#ffe082' el.textContent = 'Highlighted by Example Plugin!' } }, greet: function(options) { var el = options.element if (el) { el.textContent = 'Hello from Example Plugin!' } } }
FTML Usage:

Use your plugin in FTML as shown below:

FTML <html> <head> <script src="https://cdn.front.nu/nightly/front.js" plugin="example" conf="debug:true"></script> <!-- Include example.js in plugins folder --> </head> <body> <div example--highlight></div> <h2 example--greet></h2> </body> </html>
Note: The attribute example--highlight maps to app.plugin.example.highlight. You can define as many plugin methods as needed.

Usage Details

  • Plugin Loading: List plugin name(s) in the plugin attribute of your <script src="front.js"> tag. Place your plugin file in the plugins/ directory.
  • Method Mapping: Any attribute in the form pluginName--methodName invokes app.plugin.pluginName.methodName for that element.
  • Autoload: If defined, __autoload is called once when the plugin loads, before attribute methods.
  • Options Object: Each method receives an options object, usually with element and name.

Best Practices

  • Use __autoload for setup or initialization logic.
  • Always use Front API functions instead of direct DOM changes.
  • Keep plugin methods focused and reusable; avoid side effects outside the provided element.
  • Use descriptive and short method names (e.g., highlight, greet).
  • Always check that options.element exists before manipulating the DOM.
  • Document your plugin's attributes and expected behaviors for maintainability.
