This guide explains how to use Front’s Data module to fetch, bind, and render dynamic JSON data directly into HTML elements.

Overview

The Data module allows you to fetch remote data, bind it to DOM elements, iterate over lists, filter, sort, merge multiple responses, refresh periodically, and cache results.

Basic Usage

Include the Data module via Front and add data-* attributes to your elements:

FTML <div data-src="/api/users"</div> <ul data-iterate="dynamic-data-rendering.features"> <li> <span data-get="key"></span> </li> </ul>

Supported Features

  • Binding values: data-get, data-set
  • Iterating lists: data-iterate
  • Filtering arrays: data-filteritem, data-filterkey
  • Sorting arrays: data-sort, data-sortorder
  • Merging multiple API responses: data-srcjoin, data-merge
  • Periodic refreshing: data-interval
  • Caching responses with TTL: data-ttl (local or session storage)
  • Triggering API actions: GET, POST, DELETE
  • Binding data to other elements: data-bind

Advanced Example

Fetch users and orders, merge responses, filter, sort, and iterate:

FTML <div data-src="/api/users" data-srcjoin="/api/orders" data-merge="users;orders" data-filteritem="status:'active'" data-sort="name"></div> <ul data-iterate="users"> <li> <span data-get="name"></span> - <span data-get="orderCount"></span> </li> </ul>

Best Practices

  • Always use data-* attributes instead of direct DOM manipulation for dynamic data.
  • Use data-interval only when needed to avoid unnecessary network calls.
  • Use data-ttl for caching frequently fetched data to improve performance.
  • Break complex pipelines into smaller elements to keep templates readable.
  • Test merged responses to ensure keys align correctly when using data-merge.
