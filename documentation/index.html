Front is a low-code frontend framework for building interactive, data-driven, and modular applications using a markup-first approach. It extends standard HTML with FTML (Front Text Markup Language) - a declarative layer that allows you to define structure, style, layout, interactivity, and behavior directly through attributes, without writing any JavaScript or CSS.

Unlike traditional frontend frameworks or libraries that rely heavily on complex logic, Front keeps development simple, clean, and maintainable. Everything can be expressed through markup, enabling developers and designers to work seamlessly together while producing readable and scalable code.

Key Features:

  • Markup-First Development: Define all UI elements and behavior directly in markup.
  • Interactive and Dynamic: Built-in attributes enable events, data-binding, and reactive behavior.
  • Modular Architecture: Extend markup functionality with modules or plugins.
  • Template-Driven Layouts: Reusable templates ensure consistency across application.
  • Instant Prototyping: Runs natively in the browser-no compilers or build tools required.
  • Minimal Learning Curve: HTML-like syntax makes it approachable for both designers and developers.
  • AI-Friendly: Structured declarative markup makes AI code generation easier.

Why Choose Front?

Front will help you:
  • Spend less time on repetitive setup and boilerplate code.
  • Write markup that is easy to read, maintain, and scale.
  • Improve collaboration between designers and developers.
  • Build prototypes quickly and iterate without complex logic or state management.
  • Reduce bugs with simple, declarative syntax that avoids complexity.

Use Cases

Front is perfect for:
  • Rapid application prototypes.
  • Interactive dashboards or data-driven interfaces.
  • Small to medium-scale applications with minimal complexity.
  • Projects where designers and developers need to collaborate efficiently.

Before You Start

We assume you are familiar with HTML basics. If you are completely new to markup, consider reviewing introductory HTML tutorials, e.g., ↗️ MDN Web Docs.

In particular, you should be familiar with:

  • HTML elements and tags (structure and nesting)
  • Attributes (adding extra information to elements)
  • Classes and IDs (selectors and identifiers)
  • Forms and inputs (creating input fields and handling user data)
  • Lists, tables, and containers (organizing page content)
Note: Familiarity with CSS and JavaScript is helpful but not required - Front allows you to define interactivity and styling directly through markup.
