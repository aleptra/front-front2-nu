Front is a low-code library for building interactive, data-driven, and modular applications using a markup-first approach. It extends standard HTML with FTML (Front Text Markup Language) — a declarative layer that allows you to define structure, style, layout, interactivity, and behavior directly through attributes, without writing any JavaScript or CSS.

Unlike traditional frameworks that rely heavily on complex logic, Front keeps development simple, clean, and maintainable. Everything can be expressed through markup, enabling developers and designers to work seamlessly together while producing readable and scalable code.

Key Features:

  • Markup-First Development: Define all UI elements and behavior directly in FTML.
  • Interactive and Dynamic: Built-in attributes allow events, data-binding, and reactive behavior.
  • Modular Architecture: Extend core functionality with modules and plugins without modifying your HTML.
  • Template-Driven Layouts: Reusable templates ensure consistency across your application.
  • Instant Prototyping: Runs natively in the browser — no compilers or build tools required.
  • Minimal Learning Curve: HTML-like syntax makes it approachable for both designers and developers.

Why Choose Front?

  • Reduces boilerplate code and setup time.
  • Encourages readable, maintainable, and scalable markup.
  • Simplifies collaboration between design and development teams.
  • Ideal for rapid prototyping and projects that need fast iteration without heavy logic or state management.

Use Cases

Front is perfect for:
  • Rapid application prototypes.
  • Interactive dashboards or data-driven interfaces.
  • Small to medium-scale applications with minimal complexity.
  • Projects where designers and developers need to collaborate efficiently.
