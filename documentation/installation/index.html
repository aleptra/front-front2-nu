Installation

Follow these step-by-step instructions to set up your development environment, install dependencies, and create your first application using the Front CLI. These instructions work for macOS, Windows, and Linux/Unix.

Automatic Installation

Set up development environment automatically using GIT and Front CLI tool to generate a ready-to-use boilerplate application, complete with all necessary files and folder structure.

➊ Install GIT on your system:

  • macOS
  • Windows
  • Linux/Unix

➋ Clone the project:

git clone https://github.com/aleptra/front.git

➌ Create boilerplate application

make app:create
Note: On Windows PowerShell, make is not available by default. You can install it using winget install ezwinports.make, or run commands from Git Bash or Windows Subsystem for Linux (WSL).

Manual Installation

Use the manual installation method to gain a deeper understanding of the project structure or to customize your environment beyond the default setup. Follow these steps to create a fully configured, ready-to-use application with all required files and folders.

➊ Create starter folders

In the root directory of your project, create the following folders:

macOS / Linux / Unix:

mkdir -p _templates _includes assets

Windows PowerShell:

New-Item -ItemType Directory -Force -Name _templates, _includes, assets

This will create the essential directories:

  • _templates/ – stores reusable templates.
  • _includes/ – contains shared components.
  • assets/ – holds static files.

➋ Create starter files

Next, create your pages and template files to start building your project:

macOS / Linux / Unix:

touch index.html page1.html _templates/template1.html

Windows PowerShell:

New-Item -ItemType File index.html, page1.html, _templates/template1.html

This will create the essential files:

  • index.html – a page of type Start Page.
  • page1.html – a page of type Standard page.
  • _templates/template1.html – a reusable template.

You can now open these files in your editor and start adding markup. For examples of complete markup for the Start Page, Standard Page, and Template, see the Fundamentals section.
